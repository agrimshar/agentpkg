## Key File Locations

**Entry Points:**

- `bin/cli.ts` or `cli.ts`: CLI shebang entry point (invoked as `agentpkg` command)
  - Responsibility: Parse argv, dispatch to commands, format output
  
- `index.ts`: Library entry point (default export from package.json)
  - Exports: `AgentPackage` class, `FORMAT_VERSION`, utility functions, all types
  
- `compile.ts`: Compiler sub-entry (exported as `agentpkg/compile`)
  - Exports: `compile()`, `compileAll()`, `targets` array, compiler registry

**Configuration:**

- `package.json`: npm manifest with scripts, dependencies, exports, bin entry
  - Key fields: `"main"` → `dist/src/index.js`, `"bin"` → `dist/bin/cli.js`, `"exports"` for subpath entry points
  
- `tsconfig.json`: TypeScript compiler config
  - Output: `outDir: "dist"`, Input: `src/**/*` and `bin/**/*` (covers both)
  - Target: `ES2022`, module: `commonjs`, strict: true

**Core Logic:**

- `index.ts` (~630 lines):
  - `AgentPackage` class: builder methods, serialization, deserialization
  - Helper functions: `slugify()`, `writeJson()`, `writeText()`, `sha256File()`, `zipDirectory()`, `unzipFile()`
  - Public functions: `convertFromJSON()`, `validate()`, `validateZip()`, `initScaffold()`
  - Utilities: `now()` (ISO timestamp)

- `types.ts` (~286 lines):
  - Canonical data model: `AgentIdentity`, `Memory`, `Skill`, `Tool`, `CronJob`, `Workflow`, `Integration`, `KnowledgeDoc`, `Manifest`, `Secret`, `EncryptedVault`
  - Audit types: `AuditFinding`, `AuditSummary`, `AuditSeverity`, `AuditCategory`
  - Dependency types: `DepSpec`, `Lockfile`, `ResolvedDep`
  - Compile types: `CompileTarget`, `CompileResult`
  - Constants: `FORMAT_VERSION = "1.0.0"`

- `compile.ts` (~800+ lines):
  - Registry: `COMPILERS` object with 7 target functions
  - Public API: `compile()`, `compileAll()`, `targets` array
  - Compilers: `compileClaudeCode()`, `compileCursor()`, `compileCopilot()`, `compileWindsurf()`, `compileCrewAI()`, `compileOpenAI()`, `compileAPM()`
  - Helpers: `buildMemoryBlock()`, `buildGuardrailBlock()`, `buildSkillBlock()`, `buildToolBlock()`, `w()` (write), `wj()` (write JSON)

**Secrets & Security:**

- `secrets.ts` (~240 lines):
  - Encryption: `encryptSecrets()`, `decryptSecrets()` using AES-256-GCM + scrypt
  - Vault ops: `writeVault()`, `readVault()`, `vaultExists()`, `vaultInfo()`
  - Collection: `collectSecretsFromDir()`, `collectSecretsFromEnv()`, `collectSecretsFromEnvFile()`
  - Injection: `injectSecrets()` for platform-specific secret file generation
  - Constants: Algorithm settings (SCRYPT_COST, ALGORITHM, KEY_LENGTH, etc.)

- `audit.ts` (~250+ lines):
  - Threat patterns: `HIDDEN_UNICODE`, `PROMPT_INJECTION`, `SUSPICIOUS_TOOLS`
  - Scanning: `auditFile()`, `auditDirectory()`, `auditZip()`
  - Results: `AuditResult` class with aggregation and severity filtering

**Dependency Management:**

- `deps.ts` (~180 lines):
  - Parsing: `parseDep()` converts "npm:lodash@4.17" → `DepSpec`
  - Lockfile: `Lockfile` class manages `agentpkg.lock.json` (load/save/lock)
  - Resolution: `DependencyResolver` class for tracking transitive dependencies

**Adapters:**

- `openclaw.ts` (~400 lines):
  - Conversion: `convertOpenClaw()` reads live OpenClaw directory structure
  - Mapping: `SOUL_FILES`, `MEMORY_SUBDIRS`, `SECRET_TO_INTEGRATION` constants
  - Features: Handles soul extraction, memory loading, skill/cron/subagent collection, secret discovery
  - Options: `OpenClawConvertOptions` interface for includeSessions, extractSqliteMemories, includeMedia, includeSecrets

**CLI & Testing:**

- `cli.ts` (~260+ lines):
  - Commands: `cmdInit()`, `cmdPack()`, `cmdCompile()`, `cmdValidate()`, `cmdAudit()`, `cmdVault()`
  - Helpers: `getFlag()`, `hasFlag()` for argv parsing
  - Formatting: `log()`, `success()`, `warn()`, `error()`, `info()`, `heading()` with ANSI colors

- `test.js` (~400 lines):
  - Integration tests for pack, compile, validate, audit flows
  - Uses Node.js `assert` module
  - No test framework (pure Node.js synchronous tests)

## Naming Conventions

**Files:**
- Modules: lowercase with .ts extension (e.g., `index.ts`, `compile.ts`)
- Entry point: `cli.ts` with `#!/usr/bin/env node` shebang
- Test file: `test.js` (executed post-build)
- Type definitions: `.d.ts` auto-generated in dist/

**Directories:**
- Source root: project root (no `src/` subdirectory in this case)
- Built output: `dist/` (matches tsconfig `outDir`)
- Planning docs: `.planning/codebase/`

**Classes:**
- PascalCase: `AgentPackage`, `AuditResult`, `Lockfile`, `DependencyResolver`

**Functions:**
- camelCase: `slugify()`, `initScaffold()`, `convertFromJSON()`, `encryptSecrets()`
- Private helpers: lowercase with underscore prefix not used; instead private via module scope

**Types/Interfaces:**
- PascalCase: `AgentIdentity`, `Manifest`, `CompileResult`, `EncryptedVault`
- Type unions: `AuditSeverity = "critical" | "high" | "medium" | "low"`
- Type aliases: `CompileTarget`, `DepType`

**Constants:**
- UPPER_CASE: `FORMAT_VERSION`, `ALGORITHM`, `KDF`, `LOCKFILE_NAME`
- Private pattern constants: `HIDDEN_UNICODE`, `PROMPT_INJECTION`, `SUSPICIOUS_TOOLS` (module-scoped arrays)

**Variables:**
- camelCase for all: `zipPath`, `passphrase`, `outputDir`, `agentDir`, `findValue()`

## Where to Add New Code

**New Feature:**
- Primary code: Add builder method to `AgentPackage` class in `index.ts` (e.g., `addNewComponent()`)
- Types: Define interfaces in `types.ts` (e.g., `NewComponent`, `NewComponentOptions`)
- Tests: Add test case to `test.js` using Node.js assert

**New Platform Compiler:**
- Implementation: Add compiler function `compileNewPlatform()` in `compile.ts`
- Registration: Add to `COMPILERS` object in `compile.ts`
- Helpers: Use existing `buildMemoryBlock()`, `buildGuardrailBlock()`, `w()`, `wj()` functions
- Documentation: Include platform's official file structure in comments

**New Import Adapter:**
- Implementation: Create new file (e.g., `my-platform.ts`) with `convertMyPlatform()` function
- API: Match `convertOpenClaw()` signature (accept directory path + options, return `AgentPackage`)
- Mapping: Define platform-specific constant maps for skills, memories, secrets
- Export: Add to `package.json` exports as `./adapters/my-platform`

**New CLI Command:**
- Implementation: Add `cmdNewCommand()` function in `cli.ts`
- Handler: Dispatch in main argv switch statement
- Output: Use `heading()`, `success()`, `warn()`, `error()`, `info()` for consistent formatting
- Help: Add brief usage message on error

**Utilities:**
- Shared helpers: Add to appropriate module (`index.ts` for core, `audit.ts` for security, etc.)
- Cross-module helpers: Consider adding to `types.ts` if used widely
- Private module functions: Define at top of file before exports

**Security Scanning Rules:**
- New threat patterns: Add to `HIDDEN_UNICODE`, `PROMPT_INJECTION`, or `SUSPICIOUS_TOOLS` array in `audit.ts`
- Pattern object: `{ name: string, pattern: RegExp, severity: AuditSeverity }`
- Scanning: Existing `auditFile()` loop handles pattern application automatically
